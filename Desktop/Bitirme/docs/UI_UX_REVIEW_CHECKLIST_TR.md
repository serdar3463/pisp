# PISP UI/UX Review Checklist

Bu checklist App Store kalitesine yaklaşmak için kullanılmalıdır.

## Temel Akışlar

- [ ] Kullanıcı uygulamanın amacını ilk 10 saniyede anlıyor.
- [ ] Kasa, Paylaş, QR ve Güvenlik modülleri birbirinden net ayrılıyor.
- [ ] Kasa ekranında kullanıcı aynı anda çok fazla alanla boğulmuyor.
- [ ] Paylaş ekranında teknik detaylar isteğe bağlı açılıyor.
- [ ] QR ekranında kullanıcı onay vermeden bilgi paylaşılmadığını anlıyor.
- [ ] Güvenlik ekranı panik yaratmadan koruma durumunu anlatıyor.

## Boş ve Hata Durumları

- [ ] Hiç bilgi girilmediğinde yönlendirici boş durum var.
- [ ] Hiç paylaşılabilir alan yoksa sebep açıklanıyor.
- [ ] Kamera izni reddedilirse kullanıcı ne yapacağını anlıyor.
- [ ] Geçersiz QR okutulursa sade hata mesajı var.
- [ ] Kullanım limiti dolduğunda paylaşım butonu açıklayıcı şekilde pasif.

## Form Deneyimi

- [ ] Tarih alanı format uyarısı veriyor.
- [ ] E-posta alanında e-posta klavyesi açılıyor.
- [ ] Telefon alanında telefon klavyesi açılıyor.
- [ ] Sayı alanında sayı klavyesi açılıyor.
- [ ] Hatalar alanın hemen altında görünüyor.

## Görsel Sistem

- [ ] Ana renk sistemi mavi/beyaz ve tutarlı.
- [ ] Kırmızı yalnızca gerçek risk/hata için kullanılıyor.
- [ ] Kart yoğunluğu düşük, ekranlar nefes alıyor.
- [ ] Alt navigasyon 4 ana modülle sınırlı.
- [ ] Yazılar mobil ekranda taşmıyor.

## Erişilebilirlik

- [ ] Kontrast yeterli.
- [ ] Buton metinleri eylemi açık anlatıyor.
- [ ] Dinamik font büyüdüğünde metinler taşmıyor.
- [ ] Dokunma hedefleri yeterli büyüklükte.
